🔑인증Authentication

모든 API 호출에는 access_token이 필요합니다. 요청 헤더 또는 쿼리 스트링 파라미터로 전달할 수 있습니다. All API calls require an access_token for authentication. The token can be passed via the request header or as a query string parameter.

💡
access_token은 Adbrix Remaster 콘솔에서 발급받을 수 있습니다. API 접근 권한이 필요한 경우 담당 매니저에게 문의하세요. Obtain your access_token from the Adbrix Remaster console. Contact your account manager if you need API access enabled.
🔐
access_token이 아직 없으신가요? 아래 토큰 생성 가이드를 참고하세요. Don't have an access_token yet? See the Token Generation Guide below.

헤더 인증Header Authentication

GET /api/v1/analytics/report
Host: oac.adbrix.io
access_token: {your_access_token}

쿼리 스트링 인증Query String Authentication

GET /api/v1/analytics/report?access_token={your_access_token}&appkey={appkey}

🪪토큰 생성 가이드Token Generation Guide

디파이너리 콘솔에서 API 연동용 access_token을 발급하는 방법입니다. 머신 계정을 생성하면 사람 계정과 분리된 전용 토큰을 사용할 수 있습니다. Follow these steps to generate an access_token for API integration from the DFINERY console. Creating a machine account provides a dedicated token separate from personal user accounts.

1

디파이너리 콘솔 로그인Log in to the DFINERY Console

console.dfinery.ai에 접속하여 관리자 계정으로 로그인합니다. Go to console.dfinery.ai and log in with your admin account.

💡
머신 계정 생성은 관리자(Admin) 권한이 필요합니다. 권한이 없는 경우 조직 관리자에게 문의하세요. Admin privileges are required to create machine accounts. Contact your organization admin if you don't have access.
2

Users 관리 페이지 이동Navigate to Users Management

좌측 메뉴 또는 상단에서 Account > Users로 이동합니다. Navigate to Account > Users from the left menu or top navigation.

Account → Users
3

Invite Users 클릭Click Invite Users

Users 페이지 우측 상단의 "Invite Users" 버튼을 클릭합니다. Click the "Invite Users" button at the top-right corner of the Users page.

4

머신 추가하기Add Machine Account

초대 유형에서 "머신 추가하기"를 선택합니다. 머신 계정은 사람이 아닌 시스템(서버, 스크립트 등)이 API를 호출할 때 사용하는 전용 계정입니다. Select "Add Machine" from the invitation options. Machine accounts are dedicated accounts for systems (servers, scripts, etc.) that call the API programmatically.

🤖
왜 머신 계정인가? 사람 계정의 토큰은 비밀번호 변경 시 무효화될 수 있습니다. 머신 계정은 독립적으로 관리되어 API 연동의 안정성이 보장됩니다. Why a machine account? Tokens tied to personal accounts can be invalidated when passwords change. Machine accounts are managed independently, ensuring stable API integrations.
5

관리용 이름 지정 후 저장Set a Name and Save

머신 계정의 용도를 알 수 있도록 관리용 이름을 지정합니다. (예: data-pipeline, report-api-prod) 이름 입력 후 "저장"을 클릭합니다. Enter a descriptive name to identify the purpose of this machine account (e.g., data-pipeline, report-api-prod). Click "Save".

✓

토큰 확인View Your Token

Users 목록으로 돌아가면 방금 생성한 머신 계정이 표시됩니다. 해당 행에서 "토큰보기"를 클릭하면 access_token을 확인할 수 있습니다. Return to the Users list where your newly created machine account will appear. Click "View Token" on the corresponding row to reveal your access_token.

⚠️
보안 주의: 토큰은 비밀번호와 동일한 보안 수준으로 관리하세요. 코드에 하드코딩하지 말고 환경 변수나 시크릿 매니저를 사용하는 것을 권장합니다. Security Notice: Treat your token with the same security as a password. Avoid hardcoding in source code — use environment variables or a secret manager instead.
요약Summary
콘솔 로그인Console Login → Account > Users → Invite Users → 머신 추가Add Machine → 이름 저장Name & Save → 토큰보기 ✓View Token ✓

⚠️제약 사항 & 호출 제한Constraints & Rate Limits

안정적인 연동을 위해 API 제한 사항과 제약 조건을 확인하세요. Understanding the API limits and constraints for reliable integration.

⚡
호출 제한: API 호출은 1일 2,000회로 제한됩니다. 과부하 방지를 위해 스로틀링이 적용됩니다. Rate Limit: API calls are limited to 2,000 requests per day. Throttling is enforced to prevent system overload.

주요 제약 사항Key Constraints

  • 리포트는 Remaster 콘솔에서 먼저 설정해야 API로 조회 가능Reports must first be configured in the Remaster console before API access
  • 설정된 리포트의 데이터만 조회 가능 — 새 컬럼 추가 시 리포트 수정 필요Only data from configured reports is available — new columns require report modification
  • 최대 조회 기간: 현재 날짜로부터 90일Maximum lookback period: 90 days from the current date
  • 호출 제한은 리포트 API에만 적용; 메타데이터 API는 제한 없음Rate limiting applies to Report APIs only; Metadata APIs are exempt

🎯싱글터치 어트리뷰션 리포트Single-Touch Attribution Report

플랫폼, 광고 파트너, 캠페인, 비용 모델별 설치 및 전환 지표를 조회합니다. Retrieve install and conversion metrics by platform, ad partner, campaign, and cost model.

GET https://oac.adbrix.io/api/v1/analytics/report?appkey={appkey}&report_id={report_id}&begin_date={start}&end_date={end}

파라미터Parameters

appkey required
string
Remaster 콘솔에서 발급받은 앱키Application key from the Remaster console
report_id required
string
설정된 리포트의 고유 식별자Report identifier from the configured report
begin_date required
datetime
ISO 8601 형식의 시작 일시Start datetime in ISO 8601 format
yyyy-MM-ddThh:mm:ss
end_date required
datetime
ISO 8601 형식의 종료 일시End datetime in ISO 8601 format
yyyy-MM-ddThh:mm:ss
group_by optional
string
시간 그룹핑: daily, hourly, weekly, monthlyTime grouping: daily, hourly, weekly, monthly

제공 지표Available Metrics

  • new_install_click — 클릭 어트리뷰션 기반 신규 설치New installs via click attribution
  • new_install_view_through — 뷰스루 어트리뷰션 기반 신규 설치New installs via view-through attribution
  • re_install_click — 클릭 기반 재설치Re-installs via click attribution
  • purchase_count — 총 구매 이벤트 수Total purchase events
  • total_gross — 총 매출Total revenue
  • refund_count / refund_gross — 환불 지표Refund metrics
  • ctit_* — 클릭-설치 시간 간격 (CTIT)Click-to-install time intervals

📈일반 이벤트 분석 리포트General Event Analytics Report

커스텀 이벤트 데이터를 유연한 차원(dimension)과 집계(aggregation)로 조회합니다. 커스텀 프로퍼티와 차원을 지원합니다. Track custom event data with flexible dimensioning and aggregation. Supports custom properties and dimensions for event-level data filtering.

GET https://oac.adbrix.io/api/v1/analytics/report?appkey={appkey}&report_id={report_id}&begin_date={start}&end_date={end}

주요 지표Key Metrics

  • unique_view — 이벤트를 발생시킨 고유 사용자 수Unique users who triggered the event
  • page_view — 총 이벤트 발생 횟수Total event occurrences
  • sum_value — 이벤트 값 합계Aggregated event value
📋
이벤트 분석은 싱글터치 어트리뷰션과 동일한 엔드포인트를 사용합니다. report_id에 따라 반환되는 리포트 유형이 결정됩니다. Event Analytics shares the same endpoint as Single-Touch Attribution. The report_id determines which report type is returned.

🔄유니버설 리텐션 분석 리포트Universal Retention Analytics Report

이벤트 기준 사용자 리텐션을 0일~30일 간격의 코호트 분석으로 조회합니다. Analyze user retention across events with cohort analysis spanning 0 to 30 day intervals.

GET https://oac.adbrix.io/api/v1/analytics/report?appkey={appkey}&report_id={report_id}&begin_date={start}&end_date={end}

지원 차원(Dimension)Supported Dimensions

  • daily — 일별 리텐션Retention by day
  • ad_partner — 광고 파트너별 리텐션Retention by advertising partner
  • platform — 플랫폼별 리텐션 (Android / iOS / Web)Retention by platform (Android / iOS / Web)
  • tracker_id — 트래커 링크별 리텐션Retention by tracker link

지표Metrics

  • standard_user — 기준 코호트 규모Base cohort size
  • retention_0d ~ retention_30d — 일별 잔존 사용자 수Retained users at each day interval

⚡LiveBrix 실시간 APIReal-Time API

GET https://oac.adbrix.io/api/v1/Analytics/LiveBrix/GetRealtimeAPIReport?appkey={appkey}&begin_date={start}&end_date={end}&timezone_offset={offset}

}
